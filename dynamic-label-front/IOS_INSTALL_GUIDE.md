# iOS Safari 安装 PWA 详细步骤

## 📱 当前状态

从截图看，你已经打开了分享菜单！这是正确的第一步。

## ✅ 接下来的步骤

### 步骤1: 向下滚动分享菜单

在当前的分享菜单中：
- 你看到了：共享、书签、新建标签页等选项
- **需要继续向下滚动**，找到"添加到主屏幕"选项

### 步骤2: 找到"添加到主屏幕"

向下滚动后，你应该会看到：
- **"添加到主屏幕"** 选项
- 通常有一个图标（加号或主屏幕图标）
- 这个选项可能在菜单的底部

### 步骤3: 点击"添加到主屏幕"

1. 点击"添加到主屏幕"选项
2. 会打开一个新页面
3. 可以修改应用名称（默认是"Smart Warehouse"）
4. 点击右上角的"添加"按钮

### 步骤4: 完成！

- 应用图标会出现在主屏幕上
- 点击图标即可打开，全屏运行

## 🎯 如果找不到"添加到主屏幕"选项

### 可能的原因：

1. **需要继续向下滚动**
   - iOS 分享菜单可能很长
   - 继续向下滚动，直到看到"添加到主屏幕"

2. **需要先关闭菜单重新打开**
   - 点击屏幕其他地方关闭菜单
   - 再次点击分享按钮
   - 向下滚动查找

3. **浏览器版本问题**
   - 确保使用 Safari 浏览器
   - iOS 版本需要 iOS 11.3 或更高

4. **网站未满足 PWA 要求**
   - 需要 HTTPS（你已经有了）
   - 需要 manifest.json（已经配置）
   - 需要 Service Worker（已经配置）

## 💡 快速提示

- **分享菜单图标**：方框+向上箭头
- **"添加到主屏幕"位置**：通常在分享菜单的底部
- **图标样式**：可能是一个加号或主屏幕图标

## 🔍 验证 PWA 配置

如果还是找不到，可以检查：

1. **打开开发者工具**（如果可能）
   - 查看 Console 是否有错误

2. **检查 manifest**
   - 访问：`https://你的IP:8080/manifest.json`
   - 应该能看到 JSON 配置

3. **检查 Service Worker**
   - 在 Safari 设置中允许网站使用 Service Worker

## 📝 替代方法

如果分享菜单中没有"添加到主屏幕"：

1. **使用书签方式**
   - 先添加到书签
   - 然后从书签中添加到主屏幕

2. **直接输入 URL**
   - 在主屏幕添加书签
   - 编辑书签，改为应用 URL

现在试试在分享菜单中向下滚动，应该能找到"添加到主屏幕"选项！
